Select

Tag:

<gs-select>
    <option>...</option>
</gs-select>

Description:

The <gs-select> allows you to choose from a static dropdown list of options.

Examples:

Skeleton Example:

Without attributes the <gs-select> will have the following behaviors:

  1. The first option is the default.
  2. Clicking on the control brings up a list of options.
  3. Clicking on the a option in the control closes the list and changes the option the control has selected.
Attribute [onchange=""]:

To run javascript when the value changes, use the [onchange=""] attribute.

Attribute [value=""]:

To set the default value for the <gs-select>, use the [value=""] attribute.

Option Attribute [value=""]:

When you get the value of a <gs-select>, it will use the [value=""] attribute of the <gs-option> to return. If there is no [value=""] attribute on the <gs-option>, it will return the text of the <gs-option>.

Attribute [mini]:

The [mini] attribute allows you to fit the <gs-select> in tighter spaces.

Attribute [qs=""]:

The name [qs] stands for "querystring". The querystring is the part in the URL after the "?". The [qs] attribute has two formats, each with their own use.

  • qs="key1=attribute1". This format allows you to fill any attribute with any key from the querystring. To use multiple keys from the querystring you can put a comma-separated list of these expressions (for example: qs="key1=attribute1,key2=attribute2"). As a shorthand, if the key in the querystring is the same name as the attribute you want to fill, you can omit the attribute name in the expression (for example: qs="key1="). If an expression doesn't find it's key in the querystring: it reverts the attribute that it fills to it's original value.
  • qs="key". This format will update the [value] attribute with the querystring whenever the querystring changes.
Attribute [tabindex=""]:

With the [tabindex=""] attribute, you can control what order your controls are tabbed to.

Attribute [disabled]:

With the [disabled] attribute, you can control whether or not you can use the control.

Attribute [readonly]:

The [readonly] attribute is similar to [disabled] because it prevents the value from changing. The differences are that readonly is styled differently and allows you to copy the current text from the control.
