Combo

Tag:

<gs-combo></gs-combo>

Description:

A <gs-combo> is a combination of a textbox and a selectbox. A <gs-combo> fills the need of being able to add custom values into the control.

Examples:

Skeleton Example:

A basic <gs-combo> element with list loaded from a Postgres view using the [src=""] attribute. The [src=""] attribute can contain a view name or a table name. When you select a record out of the dropdown the text from the first column will populate the textbox

Attribute [onchange=""]:

To run javascript when the value changes, use the [onchange=""] attribute.

Attribute [cols=""]:

The [cols=""] attribute is for removing columns from the data that the combobox gets from the database.

Attribute [value=""]:

The [value=""] attribute is for setting the value of the combobox.

Attribute [hide=""]:

The [hide=""] attribute is for hiding columns that you don't want the user to see but still want access to. The primary use of this attribute is to hide the ID column in such a way that it can be used to get and set the combobox but the user never sees it.

Attribute [initialize=""]:

The [initialize=""] attribute is for loading a version of the view or table with less columns in it to speed up the inital value lookup.

Attribute [where=""]:

The [where=""] attribute is for sending a postgres WHERE with the view or table in the [src=""] or [initialize=""] attributes.

Attribute [ord=""]:

The [ord=""] attribute is for sending a postgres ORDER BY with the view or table in the [src=""] or [initialize=""] attributes.

Attribute [limit=""]:

The [limit=""] attribute is for sending a postgres LIMIT with the view or table in the [src=""] or [initialize=""] attributes.

Attribute [offset=""]:

The [offset=""] attribute is for sending a postgres OFFSET with the view or table in the [src=""] or [initialize=""] attributes. The [ord=""] attribute is for making the data in the combobox clearer and the [limit=""] attribute is so that the offset has an effect.

Attribute [qs=""]:

The name [qs] stands for "querystring". The querystring is the part in the URL after the "?". The [qs] attribute has two formats, each with their own use.

  • qs="key1=attribute1". This format allows you to fill any attribute with any key from the querystring. To use multiple keys from the querystring you can put a comma-separated list of these expressions (for example: qs="key1=attribute1,key2=attribute2"). If you need an attribute to be added when a querystring key is not present, replace the "=" with "!=" (for example: qs="key1!=attribute1", attribute1 will only be added if key1 is not in the querystring). As a shorthand, if the key in the querystring is the same name as the attribute you want to fill, you can omit the attribute name in the expression (for example: qs="key1="). If an expression doesn't find it's key in the querystring: it reverts the attribute that it fills to it's original value.
  • qs="key". This format will update the [value] attribute with the querystring whenever the querystring changes.
Attribute [limit-to-list]:

The attribute [limit-to-list] forces the user to choose a value that is in the list.

Attribute [mini]:

A <gs-combo> with the attribute [mini] will have less padding and no borders.

Attribute [disabled]:

A <gs-combo> with the attribute [disabled] will not be able to be changed by the user.

Attribute [readonly]:

The [readonly] attribute is similar to [disabled] because it prevents the value from changing. The differences are that readonly is styled differently and allows you to copy the current text from the control.

Refresh On Query String Column Change:

If you want to refresh on changes to a list of columns you can put a comma seperated list of columns in the [refresh-on-querystring-values] attribute. That could look something like [refresh-on-querystring-values="parent_id,id"].

Refresh On Query String Change:

To make the <gs-combo> refresh when anything in the query string is updated, you add the attribute [refresh-on-querystring-change].

Attribute [tabindex]:

With the [tabindex] attribute, you can control what order your controls are tabbed to.
