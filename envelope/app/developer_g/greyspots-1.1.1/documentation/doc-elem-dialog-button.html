Dialog Button

Tag:

<gs-dialog-button template="..."></gs-dialog-button>

Description:

This element is used to open a dialog at the click of a button.

Examples:

Inner Template Example:

This is an example of a <gs-dialog-button> that creates a dialog using an inner template. The template element must be an immediate child of the button for this to work. This is useful for a quick and simple dialog.

"template" Attribute Example:

In this example we have a <gs-dialog-button> that opens a dialog by linking to an external <template> element. This is done by putting the <template>'s "id" in the <gs-dialog-button>'s "template" attribute. You would use this if you want to keep all of your templates together or if you want/have multiple elements/javascript functions using the template.

Event attributes Example:

There are four event attributes that you can run JavaScript in for this element.

  1. The "before-open" attribute runs before the dialog is added to the DOM. A variable can be used in this attribute is: "this", which is the button element that was clicked.
  2. The "after-open" attribute runs after the dialog has been added to the DOM. A variable can be used in this attribute is: "this", which is the dialog element that was created.
  3. The "before-close" attribute runs before the dialog is removed from the DOM. Variables in this attribute are "this" (which is the dialog element that is going to be closed) and "strAnswer" (which is the answer of the dialog).
  4. The "after-close" attribute runs after the dialog has been removed from the DOM. Variables in this attribute are "this" (which is the button element that was originally clicked) and "strAnswer" (which is the answer of the dialog).

"target" Attribute Example:

You can open a dialog and attach it to a target element. This is done by putting a selector in the "target" attribute. This could be used for lists or tooltips.

"direction" Attribute Example:

When you open a dialog and attach it to a target element you can specify a side to attach to. If the dialog doesn't have the room on the chosen side, then it wil try different sides. This is done by putting "left", "right", "up" or "down" in the "direction" attribute.

Attribute [qs=""]:

The name [qs] stands for "querystring". The querystring is the part in the URL after the "?". The [qs] attribute has two formats, each with their own use.

  • qs="key1=attribute1". This format allows you to fill any attribute with any key from the querystring. To use multiple keys from the querystring you can put a comma-separated list of these expressions (for example: qs="key1=attribute1,key2=attribute2"). If you need an attribute to be added when a querystring key is not present, replace the "=" with "!=" (for example: qs="key1!=attribute1", attribute1 will only be added if key1 is not in the querystring). As a shorthand, if the key in the querystring is the same name as the attribute you want to fill, you can omit the attribute name in the expression (for example: qs="key1="). If an expression doesn't find it's key in the querystring: it reverts the attribute that it fills to it's original value.
  • qs="key". This format will update the [value] attribute with the querystring whenever the querystring changes.
